## Changes Made

### 1. Added `exit` prop to `ViewTransition` in ThemeTransition
- **File:** `apps/web/src/components/theme/theme-transition.tsx`
- **Change:** Added `exit` prop with the same class mappings as `enter` (`light-to-dark` → `circle-in-top-center`, `dark-to-light` → `circle-in-bottom-center`). Without this, the old (outgoing) content received no class and fell back to the browser's default crossfade animation.
- **Revert:** Remove the `exit={{ ... }}` prop block (lines 15–19)

### 2. Replaced dead `::view-transition-old(.circle-out-*)` rules with correct suppression
- **File:** `apps/web/src/app/globals.css`
- **Change:** Replaced the two never-matching `::view-transition-old(.circle-out-top-center)` and `::view-transition-old(.circle-out-bottom-center)` rules (which were dead code — those class names were never assigned by the component) with `::view-transition-old(.circle-in-top-center), ::view-transition-old(.circle-in-bottom-center) { animation: none; }`. This stops the old content from crossfading, keeping it fully visible as a static background while the new content's circle clip grows on top.
- **Revert:** Replace the new rule with the original two `::view-transition-old(.circle-out-*)` blocks.
